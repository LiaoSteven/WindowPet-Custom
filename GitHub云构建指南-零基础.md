## 🚀 三步走

### 第 1 步：上传代码到 GitHub（5 分钟）

#### 方法 A：使用 GitHub 网页上传（最简单，推荐）

1. **登录 GitHub**
   - 打开 https://github.com
   - 点击右上角"Sign in"登录

2. **创建新仓库**
   - 点击右上角的 "+" 号
   - 选择 "New repository"（新建仓库）
   - 填写仓库名：`WindowPet-Custom`
   - 选择 "Public"（公开）或 "Private"（私有，随你）
   - **不要勾选** "Add a README file"
   - 点击 "Create repository"（创建仓库）

3. **上传文件**
   - 在新页面点击 "uploading an existing file"（上传现有文件）
   - 把整个 `WindowPet-0.0.9` 文件夹里的**所有文件**拖到网页上
   - 等待上传完成（可能需要几分钟）
   - 滚动到底部，点击 "Commit changes"（提交更改）

#### 方法 B：使用 Git 命令（如果你会用 Git）

```bash
cd WindowPet-0.0.9
git init
git add .
git commit -m "Initial commit with macOS bug fix"
git branch -M main
git remote add origin https://github.com/你的用户名/WindowPet-Custom.git
git push -u origin main
```

---

### 第 2 步：触发云构建（1 分钟）

1. **进入 Actions 页面**
   - 在你的仓库页面，点击顶部的 "Actions" 标签

2. **启用 Actions**（如果是第一次）
   - 如果看到提示，点击 "I understand my workflows, go ahead and enable them"

3. **手动触发构建**
   - 在左侧找到 "手动构建 macOS 版本"
   - 点击它
   - 右侧会显示 "Run workflow" 按钮
   - 点击 "Run workflow"
   - 在弹出的菜单中：
     - "选择构建目标" 选择 **"macos-m2"**（因为你是 MacBook Air M2）
   - 点击绿色的 "Run workflow" 按钮

4. **等待构建**
   - 页面会刷新，显示一个黄色的圆圈 🟡（表示正在构建）
   - 大约等待 **10-20 分钟**
   - 当黄色圆圈变成绿色勾 ✅，就说明构建成功了！

---

### 第 3 步：下载安装包（1 分钟）

1. **查看构建结果**
   - 点击刚才的构建任务（就是那个绿色勾的那一行）

2. **下载安装包**
   - 滚动到页面底部，找到 "Artifacts"（构建产物）部分
   - 你会看到两个文件：
     - **WindowPet-macOS-M2** - 这是 DMG 安装包（推荐下载这个）
     - WindowPet-macOS-M2-APP - 这是 APP 文件
   - 点击 "WindowPet-macOS-M2" 下载

3. **解压并安装**
   - 下载的是一个 ZIP 文件，双击解压
   - 会得到一个 `.dmg` 文件
   - 双击 DMG 文件
   - 把 WindowPet 拖到 Applications 文件夹
   - 完成！

---

## 📱 如何使用应用

### 第一次打开（重要！）

1. 打开"访达" → "应用程序"
2. 找到 `WindowPet`
3. **右键点击**（不是双击！）
4. 选择"打开"
5. 在弹出的警告中点击"打开"

### 添加 Corgi 宠物

1. 应用启动后，点击**屏幕顶部菜单栏**的 WindowPet 图标
2. 选择 "Setting"（设置）
3. 点击 "Pet Shop"（宠物商店）
4. 找到 "Corgi"
5. 点击添加
6. Corgi 就会在屏幕上跑了！

## ❓ 常见问题

### Q1: 上传文件太慢怎么办？

**A:**
- 可以先删除 `node_modules` 文件夹（如果有的话）
- 只上传源代码，GitHub 会自动安装依赖

### Q2: 构建失败了怎么办？

**A:**
1. 点击失败的构建任务
2. 查看红色 ❌ 的步骤
3. 截图错误信息
4. 告诉我具体的错误

### Q3: 下载的 ZIP 里没有 DMG 文件？

**A:**
- 可能是构建失败了
- 检查 Actions 页面是否显示绿色勾 ✅
- 如果是红色 ❌，说明构建失败，需要查看日志

### Q4: 我没有 GitHub 账号，还有其他办法吗？

**A:**
有两个选择：
1. 注册一个 GitHub 账号（推荐，免费且很简单）
2. 使用原项目的预编译版本（如果有的话）

### Q5: 构建要钱吗？

**A:**
- **完全免费！**
- GitHub 每月给免费用户 2000 分钟的构建时间
- 一次构建大约用 15 分钟
- 你可以构建 100+ 次，完全够用

### Q6: 可以重复构建吗？

**A:**
- 当然可以！
- 每次修改代码后，重复第 2 步和第 3 步就行
- 不需要重新上传代码（除非你本地改了文件）

## 🔄 如果你修改了代码

比如换了新的图片或者修改了配置：

1. **上传修改后的文件**
   - 在仓库页面，点击要修改的文件
   - 点击右上角的铅笔图标（编辑）
   - 或者点击 "Add file" → "Upload files" 上传新文件

2. **重新触发构建**
   - 重复"第 2 步"
   - GitHub 会用最新的代码构建

3. **下载新的安装包**
   - 重复"第 3 步"

## 💡 小贴士

1. **保存仓库链接**
   - 把你的 GitHub 仓库链接保存到收藏夹
   - 格式：`https://github.com/你的用户名/WindowPet-Custom`

2. **构建时可以做其他事**
   - 构建需要 10-20 分钟
   - 这期间你可以关闭浏览器
   - 稍后回来刷新页面查看结果

3. **构建完可以删除旧的**
   - 在 Actions 页面可以删除旧的构建记录
   - 节省空间（虽然免费用户有 500MB 存储）
